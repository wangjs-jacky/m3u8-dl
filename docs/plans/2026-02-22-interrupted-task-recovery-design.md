# 中断任务恢复功能设计

> 日期：2026-02-22

## 背景

当下载任务中途被取消、应用崩溃或服务重启后，临时目录 `.temp_segments_*` 会保留在磁盘上。用户希望能够检测并恢复这些中断的下载任务。

## 需求

1. **提示用户选择** - 检测到中断任务后显示列表让用户决定是否恢复
2. **断点续传** - 检测已下载的分片，只下载缺失的部分
3. **元数据存临时目录** - 保存任务信息用于恢复
4. **扫描默认目录** - 只扫描用户设置的下载目录

## 设计方案

### 1. 元数据文件结构

在临时目录中创建 `task_meta.json` 文件：

```json
{
  "version": 1,
  "url": "https://example.com/video.m3u8",
  "outputPath": "/Users/xxx/Downloads/video.mp4",
  "fileName": "video.mp4",
  "totalSegments": 100,
  "downloadedSegments": [0, 1, 2, 5, 6],
  "createdAt": "2026-02-22T10:00:00Z",
  "keyInfo": {
    "method": "AES-128",
    "uri": "https://example.com/key.key",
    "iv": "0x00000000000000000000000000000000"
  },
  "headers": {
    "User-Agent": "...",
    "Referer": "..."
  }
}
```

### 2. 文件存储位置

```
<下载目录>/
  └── .temp_segments_<taskId>/
      ├── task_meta.json      # 任务元数据
      ├── segments.json       # 分片下载状态（可选，大数据量时使用）
      ├── chunks/
      │   └── part_current/
      │       ├── seg_000000.ts
      │       └── seg_000001.ts
      └── previews/
```

### 4. API 设计

#### 4.1 扫描中断任务

```
GET /api/downloads/interrupted
```

**响应**：
```json
{
  "tasks": [
    {
      "tempDir": "/Users/xxx/Downloads/.temp_segments_dl_0",
      "meta": {
        "url": "https://...",
        "fileName": "video.mp4",
        "totalSegments": 100,
        "downloadedCount": 45,
        "progress": 45,
        "createdAt": "2026-02-22T10:00:00Z"
      }
    }
  ]
}
```

#### 4.2 恢复中断任务

```
POST /api/download/resume-from-disk
```

**请求**：
```json
{
  "tempDir": "/Users/xxx/Downloads/.temp_segments_dl_0"
}
```

**响应**：
```json
{
  "id": "dl_0",
  "status": "downloading"
}
```

### 6. 实现要点

#### 6.1 元数据写入时机

- 任务创建时：写入基本信息（url、fileName 等）
- 每下载一个分片后：追加到 `downloadedSegments` 数组
- 任务完成/取消时：删除元数据文件或整个临时目录

#### 6.2 断点续传实现

```typescript
// 恢复下载时
async resumeFromDisk(tempDir: string) {
  const meta = JSON.parse(fs.readFileSync(path.join(tempDir, 'task_meta.json')));

  // 读取已下载的分片
  const downloaded = new Set(meta.downloadedSegments);

  // 创建下载器时跳过已下载的分片
  for (let i = 0; i < meta.totalSegments; i++) {
    if (!downloaded.has(i)) {
      // 下载这个分片
    }
  }
}
```

#### 6.3 原子性保证

- 使用 `writeFileSync` + 临时文件重命名保证元数据写入原子性
- 每下载 10 个分片批量更新一次元数据，减少 IO

### 7. 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 元数据文件损坏 | 显示警告，提供删除临时目录选项 |
| 分片文件损坏 | 重新下载该分片 |
| 临时目录无元数据 | 跳过该目录，或提示用户手动处理 |
| 用户点击"忽略" | 记录到设置中，下次不再提示这些任务 |
